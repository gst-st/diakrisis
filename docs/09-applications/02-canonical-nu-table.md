# Каноническая ν-таблица для стандартного шеста Verum + OWL 2

*Прескриптивная Diakrisis-сторонняя спецификация ν-координаты (ранга стратификации по глубине) для каждого framework-аксиоматического пакета, поставляемого Verum в каталоге `core.math.frameworks.*`. Этот документ закрывает пробел, выявленный кросс-аудитом Verum (рекомендация №2): значения ν, поставляемые в канонической lookup-таблице Verum, операционно выбраны stdlib Verum; настоящий файл их формально предписывает.*

---

## 1. Зачем ν

Согласно Diakrisis 02-axiomatics §2 (ν-инвариант) и 06-limits/10-maximality-theorems §103.T, ν-координата артикуляции — это минимальный ординал λ такой, что α ∈ M^λ(α_0): глубина 𝖬-итерации, требуемая для достижения артикуляции из канонического основания.

Когда теорема Verum обращается к framework-аксиоме α, она наследует ν-ранг α. MSFS-координата `(Fw, ν, τ)` (согласно `core.theory_interop.coord::MsfsCoord`) — это супремум по всем зависимостям: теорема, использующая `lurie_htt` (ν=ω) и `petz_classification` (ν=2), приземляется на ν=ω.

Этот файл *прескриптивно фиксирует* ν-ранг каждого framework, поставляемого Verum, чтобы lookup-таблица в канонической lookup-таблице Verum имела формальное Diakrisis-стороннее обоснование.

## 2. Канонический шест (согласно VUVA §6.2)

| Framework | ν | Обоснование |
|---|---|---|
| `actic.raw` | 0 | Нейтральная Актика-артикуляция — базовая линия канонического примитива (Diakrisis 12-actic §1). 𝖬-итерации не требуется: это минимум решётки. |
| `lurie_htt` | ω | $(\infty, 1)$-категорная инфраструктура (HTT 1.2 — 6.2). Определения включают ω-комплетацию (последовательные копределы, диаграммы формы Δ^op), потому ν не может быть конечным. ω — *минимальный* трансфинитный ординал, достигающий $(\infty,1)$-категорной поверхности. |
| `schreiber_dcct` | ω + 2 | Когезивный ∞-топос с тройным сопряжением `∫ ⊣ ♭ ⊣ ♯` (DCCT §3.2). Два расширения над `lurie_htt`: одно для самих когезивных модальностей, одно для дифференциально-когомологической структуры над ними. |
| `connes_reconstruction` | ω | Операторно-алгебраическое ∞-представление по Конну–Шамседдину 2008. Реконструкция спектральной тройки требует ω-итерации над гильбертовыми приближениями; ν совпадает с базовой линией `lurie_htt`. |
| `petz_classification` | 2 | Конечномерная классификация матричных метрик по Петцу 1996. Ограниченный анализ на коразмерности 2 над конечномерным пространством состояний; ν доказуемо конечен. |
| `arnold_catastrophe` | 2 | Классификатор нормальных форм коразмерности ≤ 4 по Арнольду–Матеру. Аналогичная ограниченно-конечная коразмерная установка; ν=2 как и в `petz_classification`. |
| `baez_dolan` | ω + 1 | n-категорная инфраструктура по Гордону–Пауэру–Стриту 1995 + гипотеза стабилизации (Баэз–Долан). Один шаг над `lurie_htt` для расширения стабилизации; базовая n-категорная поверхность живёт на ω. |

## 3. Расширение OWL 2 (VUVA §21)

| Framework | ν | Обоснование |
|---|---|---|
| `owl2_fs` | 1 | SROIQ DL-разрешимый фрагмент по Шкотину 2019. Логика первого порядка с ограниченной модальной глубиной иерархии ролей; границы разрешимости доказывают конечность ν. ν=1 отражает единственный слой рассуждений об иерархии ролей над пропозициональной базой. |

## 4. Классификация по τ-флагу

τ ∈ {интенсиональный, экстенсиональный} согласно Diakrisis 06-limits/08-intensional-refinement. Интенсиональные основания различают информацию о путях (MLTT-срез); экстенсиональные основания отождествляют доказуемо-равные термы (ETT-срез).

| Framework | τ | Обоснование |
|---|---|---|
| `actic.raw` | экстенсиональный | Основание не выбрано; тривиально экстенсиональный. |
| `lurie_htt` | интенсиональный | Квази-категории несут информацию о путях — эквивалентности, не равенства. |
| `schreiber_dcct` | интенсиональный | Когезивные модальности — естественные преобразования между $(\infty,1)$-функторами; уважают интенсиональное равенство. |
| `connes_reconstruction` | экстенсиональный | Реконструкция спектральной тройки коллапсирует операторно-алгебраические состояния до определительного равенства; `is_set`-типизированная поверхность. |
| `petz_classification` | экстенсиональный | Классификация операторно-монотонных функций решается до определительного равенства на матричных метриках. |
| `arnold_catastrophe` | интенсиональный | Топологические катастрофные нормальные формы сохраняют информацию о путях гладкой структуры. |
| `baez_dolan` | интенсиональный | n-категорная инфраструктура полностью интенсиональна. |
| `owl2_fs` | интенсиональный | Синтаксическое дерево SROIQ DL несёт интенсиональную информацию о путях иерархии ролей. |

## 5. Пользовательские framework

Framework, отсутствующие в этой таблице, сводятся к `(0, интенсиональный)` согласно канонической lookup-таблице Verum. Прескрипция на уровне проекта осуществляется через Verum API `register_framework_coord` (VUVA §16.6, продолжение F6) после внедрения механизма курирования сообществом.

Для корпусов *публикационного качества*, цитирующих пользовательские framework, авторы должны подать дополнение к настоящему документу с выбором ν / τ и его математическим обоснованием — симметрично порядку добавления новых framework в стандартный шест в VUVA §6.2.

## 6. Формальная верификация таблицы

ν-ранг — *не* свободный выбор, а минимальный ординал, свидетельствующий глубину 𝖬-итерации framework. Для каждой строки выше столбец обоснования содержит набросок свидетеля глубины; полная формальная верификация каждой записи — Фаза 6 F2 (Verum доказывает собственные теоремы соответствия, в число которых входит корректная определённость этой таблицы).

CLI-инструмент `verum audit --coord` потребляет эту таблицу во время выполнения: когда Diakrisis-сторона обновляет ν-ранг (например, вследствие уточнённого свидетеля глубины), соответствующая запись в канонической lookup-таблице Verum обновляется синхронно, чтобы две поверхности не расходились.
